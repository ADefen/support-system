# Support System (Тикет‑система с демо‑интеграцией ИИ)

Система поддержки с автоматической классификацией заявок и генерацией черновиков ответов на базе Django. Проект демонстрирует полный цикл разработки на Python: от модели данных и URL‑маршрутизации до сервисного слоя с эмуляцией ИИ‑интеграции и подготовки демо‑данных.

## Стек технологий

* **Язык:** Python 3.13
* **Фреймворк:** Django 5.x
* **База данных:** PostgreSQL (в проекте настроена под PostgreSQL; для локальной разработки можно использовать SQLite)
* **Шаблонизация:** Django Templates
* **Интеграция с ИИ:** сервисный слой с эмуляцией LLM (ключевые слова + шаблонные ответы)
* **Инструменты:** venv, pip, git

## В рамках проекта я реализовал систему техподдержки с автоматической обработкой заявок:

* Спроектировал модель данных Ticket с полями для хранения темы, описания, статуса, а также для результатов работы ИИ (категория и черновик ответа).
* Настроил маршрутизацию и CRUD‑функционал: список, создание и детальный просмотр тикетов с правильной вложенностью URL (главный urls.py + include в приложении).
* Выделил сервисный слой (services.py) для логики классификации и генерации ответов — чтобы отделить бизнес‑логику от HTTP‑обработчиков.
* Реализовал UX для длинных текстов: в списке использовал обрезку через truncatechars, а полный черновик вывел на детальной странице — это делает интерфейс аккуратным и удобным.
* Подготовил демо‑данные через seed_data.py: скрипт корректно инициализирует Django вне веб‑запроса и наполняет базу тестовыми заявками с уже заполненными ИИ‑полями.
* Настроил админку с отображением ИИ‑полей и сделал их доступными только для чтения — чтобы избежать случайного редактирования.
* Сделал миграции и подготовил инструкцию по быстрому запуску, чтобы проект можно было развернуть в один клик.

## Как запустить (локально, SQLite — самый быстрый вариант)

1. Клонируй репозиторий и перейди в папку проекта.
   ```bash
   git clone <URL_РЕПОЗИТОРИЯ>
   cd support_system
3. Создай и активируй виртуальное окружение:
   ```bash
   python -m venv .venv
   # .venv\Scripts\activate   Windows
   # source .venv/bin/activate   macOS/Linux
4. Установи зависимости:
pip install -r requirements.txt
5. При необходимости переключись на SQLite (если в settings.py настроен PostgreSQL):
   ```bash
   DATABASES = {
       'default': {
           'ENGINE': 'django.db.backends.sqlite3',
           'NAME': BASE_DIR / 'db.sqlite3',
       }
   }
6. Примени миграции:
   python manage.py makemigrations
   python manage.py migrate
8. Создай суперпользователя для входа в админку:
   python manage.py createsuperuser
9. Наполни базу тестовыми данными:
   python seed_data.py
10. Запусти сервер:
   python manage.py runserver
11. Открой
    
Список заявок: http://127.0.0.1:8000/tickets/

Форма создания: http://127.0.0.1:8000/tickets/create/

Детальная страница заявки: http://127.0.0.1:8000/tickets/<id>/

Админка: http://127.0.0.1:8000/admin

## Если хочешь использовать PostgreSQL

1. Укажи параметры БД в settings.py:
   ```bash
   DATABASES = {
       'default': {
           'ENGINE': 'django.db.backends.postgresql',
           'NAME': 'support_system',
           'USER': 'postgres',
           'PASSWORD': 'your_password',
           'HOST': 'localhost',
           'PORT': '5432',
           }
   }
